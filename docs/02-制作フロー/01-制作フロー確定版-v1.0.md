# ディレクターFBをAIに置き換える方法

## 今ディレクターがやっていること

1. ブラウザでサイトを開く
2. PC/SPそれぞれ目視する
3. ヒアリング回答と見比べる
4. 問題を指摘する

**これは全て「スクリーンショットを見て判断している」。Vision AI（画像を見て判断するAI）で置き換えられる。**

## 5つのチェック要素と自動化方法

### ① レイアウトのずれ → 画像差分検知（自動化率: 100%）

ヘッドレスブラウザ（Playwright）でPC/SP両方のスクリーンショットを自動撮影し、同テンプレートの「正しいレイアウト」と差分比較する。ピクセル単位でずれを検知し、閾値を超えたらNG判定。

Vision AIすら不要。画像差分検知だけで完結する。

### ② 改行の違和感 → Vision AI + 文字数計算（自動化率: 90%）

Vision AIにスクリーンショットを見せて、テキストの改行位置に違和感がないかを判定させる。

判定基準:

- 単語の途中で切れている
- 1行だけ極端に短い（widow/orphan）
- 見出しが意図せず2行に割れている
- リスト項目の長さが極端にバラついている

補完: 文字数 × フォントサイズ × コンテナ幅の計算で、改行位置を事前予測し、問題がある場合は文字サイズやコンテナ幅を自動調整。

### ③ 文字位置の不備 → CSS/DOM解析（自動化率: 100%）

DOMの構造を解析し、テキストの配置（left/center/right）がセクションごとに適切かを自動判定。

例:

- ヘッダーのテキストがleft揃えなのに、直下のサブテキストがcenter → NG
- ボタン内テキストが上寄り → NG
- セクション見出しの揃えがページ内で不統一 → NG

Vision AI不要。CSSの解析だけで完結する。

### ④ 色味の不一致 → CSS色抽出 + ヒアリング照合（自動化率: 100%）

ヒアリングフォームで選択されたメインカラー/ベースカラーと、実際のCSSカラーを自動比較。

チェック内容:

- 選択した色とCSSの色のΔE（色差）が閾値以内か
- コントラスト比がWCAGガイドラインを満たしているか
- ページ全体のカラーバランス（メイン:サブ:アクセント = 70:25:5の法則）

Vision AI不要。CSS解析と色抽出だけで完結する。

### ⑤ 顧客のニーズとデザインの乖離 → Vision AI + ヒアリング突合（自動化率: 80-90%）

ヒアリングフォームの全回答をAIに渡し、生成サイトのスクリーンショットも渡して、突合する。

AIが検出する不一致の例:

- 「Q16で『価格の安さ』を強みと回答しているが、サイト上に料金情報が見当たらない」→ NG
- 「Q21でターゲットを『30代女性』としているが、デザインが男性向けに見える」→ NG
- 「Q27でCVを『LINE友だち追加』としているが、LINEボタンがフッターにしかない」→ NG
- 「参考サイトのテイストは高級感だが、生成サイトがポップ系になっている」→ NG

## チェック要素まとめ

| チェック項目 | 方法 | 自動化率 |
| --- | --- | --- |
| レイアウトずれ | 画像差分検知 | 100% |
| 改行の違和感 | Vision AI + 文字数計算 | 90% |
| 文字位置の不備 | CSS/DOM解析 | 100% |
| 色味の不一致 | CSS色抽出 + ヒアリング照合 | 100% |
| ニーズとの乖離 | Vision AI + ヒアリング突合 | 80-90% |

**総合: 95%はAIで自動化可能。**

残り5%（本当に主観的な判断）は、初期段階でディレクターがAIチェック結果をレビューし、AIが見落とした問題を記録。そのデータを蓄積してAIの判定精度を上げていく。3ヶ月でディレクターレビューは不要になる見込み。

# ディレクターの役割転換

## 方針: 「制作のQAチェッカー」から「納品後の顧客接点」へ

AIが品質チェックを引き受けることで、ディレクターの時間が大幅に空く。ただしディレクターの本当の価値は「品質を見る目」ではなく「顧客のニーズを汲み取る力」。これは消えない。

プロパゲートの最大の穴は「納品後の4,000社が放置されていること」。これが解約率2.5%の根本原因。ディレクターがこの穴を埋める。

## ディレクターの仕事: 2つの柱

### 柱①: 新規制作における顧客接触（継続）

品質チェックはAIに移行するが、**顧客が自分のニーズを言語化できない時に汲み取る役割は残る。** これはAIには代替できない。

具体的には:

- 電話やZoomで説明したい顧客への対応
- 3パターン提示時に「どれもピンと来ない」という顧客へのヒアリング
- アノテーターを使えない顧客（20%）の修正指示の汲み取り

### 柱②: 納品後の顧客接点（新規）

## 新しいディレクターの仕事

| 仕事 | 内容 |
| --- | --- |
| **月次メールの反応対応** | 顧客が「詳しく聞きたい」を押した時の対応。AIの提案を噌み砕いて顧客に説明 |
| **解約リスク顧客のフォロー** | メール未開封・反応なしが続く顧客にプロアクティブに連絡。「最近どうですか」の一声 |
| **知見DBの構築** | 「この業種にはこれが効く」という暗黙知を体系化。松岡が初期主導、ディレクターが肉付け |
| **提案品質のスポットチェック** | AIが生成した提案の一部をサンプルレビュー。AIの判定精度を上げるフィードバックループ |
| **顧客の声の収集** | 「No」を押した顧客の理由を深掘り。提案が的外れか、タイミングか、そもそもメールを見ていないか |

# ヒアリングフォームの改善点

回答から判明した課題と対策:

| 課題 | 対策 |
| --- | --- |
| 基本情報（電話番号・代表者名・住所）がプロンプトに反映されないことがある | プロンプトに基本情報セクションの全項目を必ず渡すよう修正 |
| FV画像が来ないとAI生成画像になり「AI感」が強くなる | フォームでFV画像を**必須項目**にする。「画像がない場合はフリー素材を使用します」と明示 |
| 参考サイトURL（Q29/Q32）の回答率は70%と高い | この回答を解析して自動で反映するパイプラインを構築（新機能） |

# 「参考サイトあり」フローの詳細

全体の70%がこのフローを通る。

| ステップ | 内容 | 所要時間 |
| --- | --- | --- |
| 1 | 参考サイトのURLを受け取る | — |
| 2 | AIが参考サイトを自動解析（セクション構成・カラー・テイスト・CTA配置） | 数分 |
| 3 | 参考サイトの構造 + ヒアリング回答 + CVR最適パターンを掛け合わせて1サイト生成 | 20-30分 |
| 4 | AIが品質チェック（5項目） | 数分 |
| 5 | 問題があればAIが自動修正 | 数分 |
| 6 | 顧客に提示 | — |
| 7 | 微調整（あれば） | 数分〜数時間 |
| 8 | 公開 | — |

**CVR観点でのガード**: 参考サイトの構造をそのまま丸コピーしない。CTA配置・フォーム導線・ファーストビューの訴求はCVRデータに基づいて最適化する。顧客が求めているのは「この雰囲気」であって、構造の完コピではない（回答で裏付け済み）。

**「言語化できない」問題の解決**: 3つ並べたら「これ」と指させる。言葉で説明する必要がない。
