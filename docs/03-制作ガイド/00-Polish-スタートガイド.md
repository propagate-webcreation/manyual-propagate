# Section 1　入社初日の準備

業務を開始する前に、このセクションの手順をすべて完了させてください。

途中でわからなくなったら、遠慮なく島村育朗 or 山田友梨にSlackでメンションして確認してください。

## 1-1　Slackの招待を受ける

Slackはプロパゲートのメインコミュニケーションツールです。業務連絡・報告はすべてSlackで行います。

### 招待メールを受け取る

まず、自分のメールアドレスを島村育朗 or 山田友梨に伝えてください。招待メールが送信されます。

1. 自分のメールアドレスを島村 or 山田に伝える
2. 招待メールが届くまで待つ（迷惑メールフォルダも確認）
3. 「今すぐ参加する」ボタンをクリック
4. 氏名・パスワードを設定してアカウント作成
5. ログインできることを確認

### Slackアプリのインストール（推奨）

ブラウザ版でも使用できますが、通知が届きやすいのでアプリを推奨します。

1. [https://slack.com/downloads](https://slack.com/downloads) にアクセス
2. 自分のOSに合ったインストーラーをダウンロード
3. インストール後、アプリを起動してログイン
4. ワークスペース「株式会社プロパゲート」が表示されることを確認

## 1-2　Slackワークスペースの初期設定

### プロフィールの設定

1. Slack左上の自分の名前をクリック
2. 「プロフィールを編集する」を選択
3. 氏名・表示名（フルネーム）を入力して保存

### 必須チャンネルへの参加

以下のチャンネルに参加していることを確認してください。

| チャンネル名 | 用途 | engineering-polish-general | 全体への連絡・情報共有を行うメインチャンネル |
| --- | --- | --- | --- |
| engineering-polish-質問の場 | 作業中の疑問を気軽に質問するチャンネル | 本社-勤怠 | 毎日の業務開始（開店）・終了（閉店）の報告 |
| 制作-ディレクターフィードバック | ディレクターからのFBを受け取るチャンネル |  |  |

### 出退勤の報告方法

毎日の業務開始・終了は「本社-勤怠」チャンネルに投稿します。

- 【業務開始時】開店します。
- 【業務終了時】閉店します。

## 1-3　Web Creationアカウントの連携（Vercel・GitHub）

Web Creationはサイト制作・管理に使用するプラットフォームです。

### ① Vercelへのログイン

Vercelはサイトを公開（デプロイ）するためのサービスです。

1. [https://vercel.com](https://vercel.com) にアクセス
2. 「Log In」をクリック
3. メールアドレス [webcreation@propagateinc.com](mailto:webcreation@propagateinc.com) でログイン
4. 認証コードは鈴木貴登さんにSlackでメンションして確認
5. プロパゲートのチームに参加できているか確認

> ⚠️ Vercelは会社共有アカウントのため、認証コードは必ず鈴木貴登さんに確認してください。
> 

### ② GitHubアカウントの作成

GitHubはコードを管理・共有するサービスです。

1. [https://github.com](https://github.com) で「Sign up」
2. メールアドレス・パスワード・ユーザー名を入力
3. 確認メールで認証を完了
4. 作成したGitHubユーザー名を島村 or 山田にSlackで伝える

### ③ GitHubリポジトリへの招待を承認

1. 招待メールの「View invitation」をクリック
2. 「Accept invitation」をクリック
3. プロパゲートのGitHub Organizationにアクセスできることを確認

### ④ GitHub CLIによる認証

ターミナルからの操作を可能にするための認証設定です。

**インストール:**

- Mac: `brew install gh`
- Windows: [https://cli.github.com](https://cli.github.com) からダウンロード

**認証:**

```
gh auth login
```

質問には以下のように回答してください：

| 質問 | 選択する答え | What account do you want to log into? | [GitHub.com](http://GitHub.com) |
| --- | --- | --- | --- |
| What is your preferred protocol? | **HTTPS**（SSHではなくこちら） | Authenticate Git with your GitHub credentials? | Yes |
| How would you like to authenticate? | Login with a web browser |  |  |

ブラウザが開き、「Authorize」をクリック。「Logged in as （ユーザー名）」と表示されれば完了です。

### ⑤ Gitのユーザー情報を会社設定にする

```
git config --global user.email "info@propagateinc.com"
git config --global user.name "propagate1"
```

> ⚠️ **重要**: 個人のGitHubアカウントの情報は使わない。コミットには必ず「propagate1」が表示されるようにしてください。
> 

## 1-4　ツールのインストール

### ① Cursor（コードエディタ）

Cursorはサイトのコードを確認・修正するためのエディタです。

1. [https://cursor.com](https://cursor.com) からダウンロード
2. インストール後、起動してGitHubアカウントでログイン

> 💡 基本操作: ファイル検索 Cmd+P / 保存 Cmd+S / ターミナル Terminal → New Terminal
> 

### ② Typeless

コード入力・作業を効率化するツールです。Google検索で「Typeless」を検索し、ダウンロード・インストールしてください。

### ③ Node.js

サイトをローカルで確認するために必要です。

1. [https://nodejs.org](https://nodejs.org) からLTS版をダウンロード
2. インストール後、ターミナルで確認: `node -v` / `npm -v`

### ④ Git

- **Mac**: `xcode-select --install`
- **Windows**: [https://git-scm.com/download/win](https://git-scm.com/download/win) からダウンロード
- 確認: `git --version`

# Section 2　毎日の業務開始・終了フロー

毎日の業務はここから始まります。

## 2-1　業務開始時

1. Slack「本社-勤怠」で「開店します」と投稿
2. 稼働記録シートに業務開始時間を入力
3. 制作進行シートの「ポリッシュ用キュー（締切順）」フィルタビューで対応案件を確認
4. 取り掛かる案件を島村 or 山田にSlackでメンションして報告

## 2-2　業務終了時

1. Slack「本社-勤怠」で「閉店します」と投稿
2. 島村 or 山田にSlackで業務終了を報告
3. 稼働記録シートに業務終了時間・休憩時間を入力

## 📌 案件の取り掛かり・完了ルール

- 新たに案件に取り掛かる際 → 島村 or 山田にSlack報告 + **制作進行シートのR列（active_polish）に自分の名前を入力**
- 案件が完了した際 → 島村 or 山田にSlack報告 + **M列ステータス変更 + R列の名前をクリア**

> 💡 **シート操作の詳細**（キューの見方・ステータス変更・提出前チェック）は [2. ポリッシュメンバー運用マニュアル](https://www.notion.so/2-{NOTION_PAGE_ID}?pvs=21) を参照してください。
> 

最初の10時間は「早さ」より「手順通りに正確にできること」を優先。

1件あたり1時間程度を目安にしてください。

# Section 5　サイト修正フロー（Git操作）

M列が「**デモサイト修正中**」または「**スマホ最適最終修正中**」の案件が対象です。

## 5-1　サイトデータを自分のPCに取り込む

### 初めてその案件を担当する場合（クローン）

```
git clone （リポジトリのURL）
```

リポジトリURLは制作案件シートのG列で確認できます。

### すでに自分のPCにデータがある場合（最新化）

作業前に必ず最新のコードを取り込んでください。

```
git pull origin main
```

## 5-2　ローカルでサイトを起動

Cursorのターミナルで以下を実行：

```
npm install
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開くとサイトが表示されます。

> 💡 `npm run dev` を止めるにはターミナルで `Ctrl + C` を押します。
> 

## 5-3　コードを修正

- Cursorでファイルを開いて該当箇所を修正
- ファイルを保存（Cmd+S）
- [localhost:3000](http://localhost:3000) で修正内容を確認（ページを再読み込み）
- **PC表示・スマホ表示の両方を確認**

## 5-4　修正をGitHubにプッシュ（コマンドの順番厳守）

```
git add .
git commit -m "修正内容の説明"
git commit --allow-empty -m "deploy"
git push origin main
```

> ⚠️ **コマンドの順番を絶対に守ること**
> 

> 正しい順序: git add . → git commit -m → git commit --allow-empty → git push
> 

> 💡 コミットメッセージの例:
> 

> 「トップページのファーストビュー画像を差し替え」
> 

> 「会社概要ページの電話番号を修正」
> 

## 5-5　Vercelで本番サイトへの反映を確認

プッシュ後、数分でVercelに自動デプロイされます。

- 制作進行シートのAI列（本番URL）またはAG列（テストURL）をブラウザで開く
- 修正内容が正しく反映されているか確認
- **PC表示・スマホ表示の両方を確認**
- 修正した箇所以外でデザイン崩れや不具合（デグレ）が出ていないか確認

> 💡 通常1〜3分で反映されます。5分以上経っても反映されない場合は島村 or 山田に相談してください。
> 

## 5-6　修正完了後のシート操作

> 👉 **シート操作の詳細は運用マニュアル セクション④⑧を参照**
> 

修正が完了したら、運用マニュアルに従って以下を実行してください：

# Section 7　Gitコマンド クイックリファレンス

よく使うGitコマンドの一覧です。

| コマンド | 意味・用途 | `git clone （URL）` | リポジトリを初めて自分のPCにコピーする |
| --- | --- | --- | --- |
| `git pull origin main` | 最新のコードをリモートから取り込む（修正前に必ず実行） | `npm install` | 必要なパッケージをインストールする |
| `npm run dev` | ローカルでサイトを起動（[localhost:3000](http://localhost:3000)で確認） | `git add .` | 変更ファイルをすべてステージング |
| `git commit -m "メッセージ"` | 変更内容をコミット（メッセージは自分で記述） | `git commit --allow-empty -m "deploy"` | Vercelへのデプロイをトリガーする空コミット |
| `git push origin main` | GitHubにプッシュしてVercelに自動デプロイ |  |  |

## 🔁 毎回の修正作業の正しいコマンド順序

1. `git pull origin main` ← 作業前に必ず最新化
2. （Cursorでコードを修正・保存）
3. `git add .`
4. `git commit -m "修正内容の説明"`
5. `git commit --allow-empty -m "deploy"`
6. `git push origin main`
7. VercelのURLを開いて反映を確認
