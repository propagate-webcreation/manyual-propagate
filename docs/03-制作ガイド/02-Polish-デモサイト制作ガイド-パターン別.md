# Polish デモサイト制作ガイド（パターン別対応手順）

> 📋 **このページの目的**
> 

> デモサイト制作時に、ヒアリングシート以外の経路で顧客情報が提供されるケース（パターンB/C）に、ポリッシュメンバーが**自力で対応できる**ようにするためのガイドです。
> 

> **従来**: パターンB/C → 即エスカレーション（後回しにして渡辺/大林を待つ）
> 

> **改訂後**: パターンB/C → まずこのガイドで自己対応を試み、解決できない場合のみエスカレーション
> 

---

# 案件の3パターン

| パターン | 状況 | J列 | K or L〜Q列 | 対応 |
| --- | --- | --- | --- | --- |
| **A（通常）** | ヒアリングシート経由で情報が揃っている | ✅ | ✅ | 既存フロー通り |
| **B（構成あり）** | 顧客が独自のサイト構成を提供している | ✅ | ❌ or 不一致 | **構成変換手順**で対応 |
| **C（断片情報）** | LINE画像・手書きメモ等で情報が散在 | ✅ or ❌ | ❌ | **構成生成手順**で対応 |

---

# パターン判定フロー

デモサイト制作を開始する**前に**、以下のフローで案件パターンを判定してください。

## ① J列を確認する

- → データあり → **②へ**
- → データなし or 明らかに不完全 → **④へ**

## ② K列（BASICの場合）または L列（STANDARD/ADVANCEの場合）を確認する

- → データあり → **パターンA（通常）** → 既存の制作フローへ進む
- → データなし or 明らかに不完全 → **③へ**

## ③ AF列・LINE・メール・資料(S列)を確認する

- → 顧客からの構成情報あり（セクション順序＋各セクションの内容が明記されている）
    
    → **パターンB（構成あり）** → 下記「パターンB対応手順」へ
    
- → 顧客からの情報はあるが断片的（画像・メモ・テキスト等がバラバラ）
    
    → **パターンC（断片情報）** → 下記「パターンC対応手順」へ
    
- → 情報が見当たらない → **エスカレーション**

## ④ J列が空の場合

- AF列・LINE・メール・資料(S列)に顧客情報があるか確認
- → あり → **パターンC（断片情報）**（J列の生成から対応）
- → なし → **エスカレーション**

> ⚠️ **パターンBとCの見分け方**: 顧客提供情報に「**セクション順序＋各セクションの内容**」が含まれていればB。「載せたい情報はあるがページ構成の指示はない」場合はC。
>

# パターンB 対応手順：顧客構成の変換

> 📍 **前提**: 顧客がサイト構成を明確に提供している。J列（デザイン指示書）は存在する。
> 

> **所要時間目安**: 15〜30分
> 

## B-1: 顧客構成情報の収集

以下のすべてを確認し、構成情報を集める：

- AF列（先方要望）
- LINE（先方のライン）
- メール
- S列（資料）
- BJ列（先方サイトURL）

## B-2: 構成データの変換（Gemini使用）

**新しいGeminiチャットを開く（タブ1）**

以下のプロンプトをペーストし、`[ここに顧客提供の構成情報をペースト]` 部分に収集した構成情報を貼り付ける。

**画像の場合はそのままGeminiに添付する**（Geminiが読み取る）。

```
以下は顧客から提供されたサイト構成情報です。

---
[ここに顧客提供の構成情報をペースト（テキスト、画像、ファイル内容等）]
---

この情報をもとに、以下のフォーマットでサイト構成を整理してください。
顧客の構成・順序・内容を100%漏れなく反映し、勝手な追加・削除・順序変更をしないこと。

【出力フォーマット】
各セクションについて以下の形式で出力：

### セクションN: [セクション名]
【テキスト情報】
- 見出し: [顧客指定のテキスト]
- 本文: [顧客指定のテキスト]
- リスト項目: [ある場合]
【画像・配置の説明】
- [顧客が指定した画像の配置・内容]
- ※画像指定がない場合は「画像プレースホルダー: [セクション内容に適した説明]」と記載

注意事項：
- 顧客が指定していないセクションを追加しない
- テキストは顧客原文をそのまま使用（誤字修正のみ可）
- 「よくある質問」セクションがある場合はアコーディオン形式と注記
- 「所在地」セクションがある場合はGoogleマップ埋め込みと注記
- 図解・表がある場合はコーディングで再現する旨を注記
```

## B-3: STANDARD/ADVANCEの場合 — ページ分割

> ⚠️ **STANDARD-CP / ADVANCE-CP の場合のみ実施。** BASICの場合はスキップしてB-4へ。
> 

同じGeminiチャットで続けて以下を入力：

```
先ほど整理したサイト構成を、以下の区分に分割してください。

1. 【TOPページ構成】: TOPページのセクションのみ
2. 【サイト全体構成】: 全ページの一覧表（ページ名・概要・主要セクション）
3. 【下層ページ構成】: ページごとに分けて出力
   - サービス・料金系ページ
   - 会社概要系ページ
   - お問い合わせ系ページ
   - その他ページ

各区分は先ほどと同じフォーマットで出力すること。
```

## B-4: 変換結果の確認

Gemini出力を確認し、以下をチェック：

- [ ]  顧客指定のセクション順序が保持されているか
- [ ]  テキスト情報が原文のまま反映されているか
- [ ]  勝手なセクション追加がないか
- [ ]  抜け落ちたセクションがないか

> ✅ **問題なければ → 通常の制作フローへ進む**
> 

> B-2/B-3の出力を、K列（BASIC）または L〜Q列（STANDARD/ADVANCE）の代わりにGemini Canvasにペーストする。
>

# パターンC 対応手順：断片情報からの構成生成

> 📍 **前提**: 顧客情報がLINE画像・手書きメモ・ドキュメント等に散在。体系的な構成指示がない。
> 

> **所要時間目安**: 30〜60分
> 

## C-1: 情報の収集

以下のすべてを確認し、顧客情報を集める（B-1と同じ確認先）：

- AF列（先方要望）
- LINE（先方のライン）
- メール
- S列（資料）
- BJ列（先方サイトURL）
- ヒアリングシート（E列のリンク先）

## C-2: 情報整理（Gemini使用）

**新しいGeminiチャットを開く（タブ1）**

```
以下は顧客から様々な形式で提供された情報です。
すべての情報を漏れなく読み取り、整理してください。

---
[ここに顧客提供の全情報をペースト（テキスト・画像すべて）]
---

以下の項目に分類して整理してください：

1. 【基本情報】会社名/サービス名、業種・業態、所在地・連絡先、営業時間等
2. 【サービス・商品情報】サービス内容、特徴・強み、料金等
3. 【ターゲット・訴求】ターゲット層、強調ポイント
4. 【素材情報】提供画像の内容リスト、参考サイトURL、希望の色味・雰囲気
5. 【その他】上記に分類できない情報

注意：提供情報に含まれる内容のみ記載（推測で追加しない）。画像内テキストも読み取ること。
```

## C-3: デザイン指示書の生成（J列が空の場合のみ）

> 💡 **J列にデータがある場合はスキップしてC-4へ。**
> 

→ 詳細プロンプトは「[パターンB/C テンプレート集](templateページへのリンク)」を参照

## C-4: サイト構成の生成（K列 or L〜Q列用）

**新しいGeminiチャット（タブ2）**で以下を実行。

```
以下の整理済み情報をもとに、[プラン名] のサイト構成を作成してください。

---
[C-2の出力をペースト]
---

プラン: [G列のプラン名]

【出力ルール】
- 提供情報に含まれるテキスト・内容を最大限活用する
- 情報が不足しているセクションはダミーテキストで補完し [※ダミー] と注記する
- 各セクションは以下のフォーマットで出力：

### セクションN: [セクション名]
【テキスト情報】
- 見出し: ○○○
- 本文: ○○○○
【画像・配置の説明】
- 画像プレースホルダー: [説明]

【プラン別のセクション構成ガイドライン】
■ BASIC-LP: メインビジュアル → 課題提起 → サービス紹介 → 特徴・強み → 料金 → お客様の声 → よくある質問 → CTA → 会社概要 → フッター
■ BASIC-CP: メインビジュアル → 事業紹介 → サービス一覧 → 会社概要 → アクセス → お問い合わせ → フッター
■ STANDARD-CP: TOPページ + サービス・料金 + 会社概要 + お問い合わせ + 実績・お客様の声 + ブログ（任意）
■ ADVANCE-CP: TOPページ + サービス詳細×4 + 会社概要 + お問い合わせ + 実績 + お客様の声 + FAQ + 採用情報 + お知らせ

注意：
- 「よくある質問」→ アコーディオン形式
- 「所在地」→ Googleマップ埋め込み
- 図解・表 → コーディングで再現（画像禁止）
- 対応エリア → 画像・イラスト生成禁止
```

## C-5: STANDARD/ADVANCEの場合 — ページ分割

B-3と同一手順を実施。

## C-6: 出力の確認

Gemini出力を確認し、以下をチェック：

- [ ]  顧客提供のテキストが反映されているか
- [ ]  [※ダミー] 箇所が最小限か（顧客情報で埋められる箇所がダミーになっていないか）
- [ ]  プランのページ数/構成に合っているか

> ✅ **問題なければ → 通常の制作フローへ進む**
> 

> C-3/C-4/C-5の出力を、J列・K列 or L〜Q列の代わりにGemini Canvasにペーストする。
>

# エスカレーション基準（改訂）

以下の場合は**#制作-ai_質問の場**にエスカレーションする：

| 状況 | 対応 |
| --- | --- |
| 顧客情報がどこにもない（AF列・LINE・メール・S列すべて空） | エスカレーション |
| パターンC対応でGemini出力の品質が著しく低い（[※ダミー]だらけ） | エスカレーション |
| 顧客の要望がプラン仕様を大幅に超えている（BASIC案件で10ページ以上等） | エスカレーション |
| 技術的な問題（Gemini Canvas/Manus/Vercel等のエラー） | エスカレーション |

**エスカレーション時の報告テンプレート:**

```
【エスカレーション】
- レコード番号: ○○
- パートナー名: ○○
- パターン: B / C / その他
- 詰まりポイント: [具体的に何が解決できないか]
- 試したこと: [STEP 0-B/C のどこまで実施したか]
```
