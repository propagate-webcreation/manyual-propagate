# 1. ディレクターの役割

新運用でのディレクターは**品質チェック・FB**と**初回の顧客提出**に集中します。

1. **2つのチェックポイント**でのデザイン品質チェック・FB
    - **デモサイト**: OKなら「ディレクターチェック完了」にステータス変更 → **営業がデモ実施・顧客に提出**
    - **スマホ最適**: OKなら**ディレクターが顧客に提出**
2. **ポリッシュ・Wixデザイナーからのエスカレーション対応**（判断・サポート）
3. **公開フェーズ**でのドメイン接続・本番化

> 💡 ポリッシュメンバーへのタスク割り振り（アサイン）は不要です。ポリッシュはキュー方式で上から順に着手します。
> 

## ディレクター自動振り分け

M列が「デモサイト制作中」に変更された際、I列（ディレクター）が空欄の場合、GASが自動でディレクターを割り当てます。手動でのアサインは不要です。

## ① デモサイト制作完了（AI制作）

> 📍 **いつこのステータスになるか**: ポリッシュがデモサイト制作を完了し、M列を「デモサイト制作完了」に変更したときです。
> 

**やること:**

1. **AJ列**（test_url）のVercelテストURLを開く
2. デモサイトのデザイン品質をチェック
3. **AH列**（ヒアリングシートURL）と照合し、顧客要望が正しく反映されているか確認
4. 品質に問題がなければ → **OKの操作** / FBが必要なら → **FBありの操作**

**OKの場合:**

| # | 列 | 操作 |
| --- | --- | --- |
| 1 | **M列** | 「**デモサイトディレクターチェック完了**」に変更 |

> 💡 **顧客への提出はディレクターではなく営業が行います。** 営業がデモを実施し、デモサイトを顧客に提出します。ディレクターはステータス変更のみ。
> 

**FBありの場合（2026-04-10〜 Bot駆動フロー）:**

フィードバックは**サイトアノテーターに書き込む**だけでOK。Botが自動反映します。

**通常時の操作（パターンA / 推奨）:**

| # | 列 / ツール | 操作 |
| --- | --- | --- |
| 1 | **サイトアノテーター** | 修正依頼を一流し書き込む |
| 2 | **M列** | ⚠️ **必ず**「**デモサイト評価中**」に変更（Botが自動修正を開始） |

> 💡 Botが自動修正完了後、ステータスは自動で「デモサイトディレクターFB修正中」に遷移し、ポリッシュが残りを手動修正します。ポリッシュ完了後は「デモサイト制作完了」に戻り、ディレクターが再チェックします。
> 

**緊急時の操作（パターンB / 例外）:**

以下の場合のみ、Botをスキップして直接「デモサイトディレクターFB修正中」に変更:

- **当日中にポリッシュ対応を完了させなければならない場合**（Bot処理を待てない）
- **人間の目でないと判断が難しい場合**（Botでの対応が難しいケース）

> 📢 **注意点・共有事項がある場合のみ**: **#制作-ai_ディレクターfb** にキントーン番号＋案件名を付けて投稿。通常のFBはアノテーターのみでOK。
> 

> ➡️ **次の工程**:
> 

> - **OK**: M列 =「デモサイトディレクターチェック完了」（ボール: **営業** → 営業がデモ実施・顧客に提出）
> 

> - **FB通常**: M列 =「デモサイト評価中」（ボール: **AI Bot** → Bot自動修正後「デモサイトディレクターFB修正中」に自動遷移 → ポリッシュが残りを手動修正 → 「デモサイト制作完了」に戻る → **ディレクターが再チェック**）
> 

> - **FB緊急**: M列 =「デモサイトディレクターFB修正中」（ボール: **ポリッシュ** → 修正後「デモサイト制作完了」に戻る）
>

## ③ 公開（AI制作）

> 📍 **いつこのステータスになるか**: スマホ最適が顧客承認され、PSがM列を「公開」に変更したときです。
> 

**やること:**

1. 顧客にドメイン情報を確認する（ドメイン接続案内）
2. ドメイン接続作業を実施
3. 本番サイトの最終確認（リンク切れ・表示崩れ等）

**完了時の操作:**

| # | 列 | 操作 |
| --- | --- | --- |
| 1 | **AL列** | 本番URL（production_url）を記入 |
| 2 | — | 顧客に公開完了を連絡 |

> ➡️ **次の工程**: 公開完了 → 案件クローズ（完了タブへ自動移動）
> 

## 完了タブへの自動移動条件

以下の**必須7条件**が全て揃うと、確認ダイアログが表示され、「はい」で完了タブに自動移動します。

**※AQ列（GA）・AR列（GSC）・AS列（Clarity）は任意です。未入力でも完了移動をブロックしません。**

| 列 | 項目 | 条件 |
| --- | --- | --- |
| AL | production_url | 本番URLが入力されている（非空） |
| AM | check_links | ✅ または チェックボックスON |
| AN | check_form_notify | ✅ または チェックボックスON |
| AO | favicon | ✅ または チェックボックスON |
| AP | domain_status | 「ドメイン接続完了」 |
| AQ | check_google | ✅ または チェックボックスON |

> ※ **AR列（GA接続）・AS列（GSC接続）・AT列（Clarity接続）は任意です。未入力でも完了タブへの移動はブロックされません。**
> 

> ⚠️ **AU列（completion_date / 完了日）に日付を入力することも必須です。** 完了日が空欄だと完了タブに移動できません。

## デモサイトのFB（AI制作）— Bot駆動

**通常手順（パターンA）:**

1. **サイトアノテーター**（修正ツール管理画面）で修正依頼を一流し書き込む
    - 修正箇所を具体的に指示（ページ名・セクション・修正内容）
    - 用語は用語集に従う
2. M列を「**デモサイト評価中**」に変更 → Botが自動修正を開始
3. Bot処理完了後、自動で「デモサイトディレクターFB修正中」に遷移され、ポリッシュが残りを手動修正
4. ポリッシュ完了後、「デモサイト制作完了」に戻る → ディレクターに再チェック依頼

**緊急手順（パターンB）:**

以下の場合のみ、Botをスキップして直接「デモサイトディレクターFB修正中」に変更:

- **当日中にポリッシュ対応を完了させなければならない場合**
- **人間の目でないと判断が難しい場合**

# 5. 上記以外のフェーズ

デモサイトのディレクターチェック・初回スマホ最適提出の2つのチェックポイント以外は、**ポリッシュメンバー（AI制作）またはWixデザイナー（Wix制作）が直接対応・顧客提出します。**

具体的には以下のフェーズはポリッシュ/Wixデザイナーが直接顧客に提出・対応します：

- デモサイト修正（顧客FBを受けての修正 → 再提出）
- スマホ最適修正（顧客FBを受けての修正 → 再提出）

**ディレクターの介入は不要です。** ポリッシュ/Wixデザイナーが判断に迷った場合のみ、エスカレーションとして対応します。

# 6. エスカレーション対応

ポリッシュメンバーからのエスカレーションは**全てディレクターが受けます。**

| エスカレーション内容 | ディレクターの対応 |  |
| --- | --- | --- |
| デザインの方向性に不安 | デザイン方針をFBし、ポリッシュに差し戻す |  |
| 顧客の修正指示が不明確 | 内容を解釈し、具体的な指示をポリッシュに出す |  |
| 技術的に対応が難しい | 代替案を検討して指示、または自ら対応 |  |
| 全面やり直しの要求 | フェーズ期限日・方針を判断し、ポリッシュに具体的な指示を出す |  |
| フェーズ期限日に間に合わない | T列のフェーズ期限日を調整。必要に応じてPSに顧客連絡を依頼 |  |
| 顧客クレーム（PS経由） | 状況を確認し対応方針を決定。必要時に営業に連携 |  |

> 💡 ポリッシュは営業に直接エスカレーションしません。全てディレクター経由です。ディレクターが「結び」の役割として介在します。
>

# 7. 業務依頼への対応

> 💡 **ポリッシュ宛ての業務依頼は廃止されました。** 制作の進行状況・対応ステータスは制作進行シートで管理し、制作に関する質問・確認事項は**ディレクター宛て**に業務依頼として届きます。
> 

## PSからの業務依頼

PSから業務依頼シート経由で連携が来るケース：

- Q列が「ディレクター」のときに顧客から連絡があった場合
- 顧客から全面やり直しの要求があった場合
- 大幅修正のフェーズ期限日相談
- 制作上の判断が必要な案件の相談
- **制作に関する質問・確認事項**（修正・スマホ最適依頼に関するもの）

## ポリッシュからの業務依頼

ポリッシュメンバーからの業務依頼は、**エスカレーション**（判断に迷う場合・技術的な問題等）として届きます。上記「エスカレーション対応」セクションに従い対応してください。
